# CABA (Capability Assurance & Boundedness Architecture)

This document defines the framework for reviewing and bounding new capabilities in Way of Pi.

## Checklist

- [ ] Capability is listed in the CABA checklist (`CABA_CHECKLIST.md`).
- [ ] Documentation for the capability includes a link to the CABA entry.
- [ ] The capability's scope is bounded to a workspace root (see `WOP_NAMESPACE.md`).
- [ ] A risk assessment is performed and approved by the product team.
